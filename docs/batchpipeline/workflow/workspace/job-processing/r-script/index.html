Rscript

#Definition

R실행환경 구성된 원격 서버 내의 R 파일을 실행한다. [Flow구성]노드 중 [Rscript]노드를 drag & drop 한 후 Property 항목을 입력한다. Property 패널의 [더보기+] 버튼을 누르면 입력가능한 전체 Property 항목을 볼 수 있다.

#Set

[setting], [scheduler], [parameter] 설정은 [워크플로우 생성] > [설정]을 참고한다.

#property

[Node Description] 작성 중인 노드명 입력

  1. appName : 앱 이름 설정
  2. R 실행 IP : IP주소 및 PORT(자동입력)
    • IP주소 : 원격서버 접속 IP
    • PORT : 원격서버 접속 PORT
  3. user : R서버 사용자 아이디 (자동입력)
  4. password : R 서버 사용자 패스워드 (자동입력)
  5. 원격서버 ID : R 서버 ID (자동입력)
    • R 서버 정보[2~5]는 "설정"->"원격서버관리" 에 등록된 원격서버 리스트 중 선택된 정보로 자동 입력된다
    • "목록열기" 아이콘을 클릭하여 기 등록된 원격서버정보를 적용한다.
  6. sourcepath : 실행할 R 코드가 저장된 R 파일 path
    • 원격서버의 Filesystem 조회 및 선택을 통해 실행하고자 하는 R 파일 경로를 지정한다.
  7. source : 실행될 R 코드 조회
    • 6번 sourcepath 경로의 R 코드 조회가 가능하다. (자동입력, 편집기능은 미지원)
  8. argument : 인수 설정 (R 실행 command)
  9. forceOK : 실패 시 강제 OK 처리 여부

#Example

숫자열을 출력하는 rscript를 원격지 R 서버에서 실행하는 예제이다. test.R 이란 스크립트를 R서버에 등록 후 "6.sourcepath"에 지정하거나, "7.source"에 직접 코딩하여 실행할 수 있다. Source Editor 버튼("7.source" 우측 첫 번째 버튼) 클릭 시 Editor가 팝업되며 보다 큰 화면에서 코드를 작성할 수 있다.

  1. 연결관리에서 원격서버를 선택한다. 선택시 2.R실행IP ~ 5.원격서버ID까지 자동입력된다.

  2. rscript를 작성한다. 또는 sourcepath에서 원격서버에 미리 생성한 rscript를 실행한다.
