subWorkflow

Definition#

다른 워크플로우를 하나의 Node로 실행 할 수 있다. [Flow구성]노드 중 [subWorkflow]노드를 drag & drop 한 후 Property 항목을 입력한다. Property 패널의 [더보기+] 버튼을 누르면 입력가능한 전체 Property 항목을 볼 수 있다.

목록 열기를 누르면 목록이 팝업 되고 이중 하나를 선택하면 워크플로우 번호가 입력 된다.

Set#

[setting], [scheduler], [parameter] 설정은 [워크플로우 생성] > [설정]을 참고한다.

property#

[Node Description] 작성 중인 노드명 입력

  1. propagateConfiguration : subWorkflow 설정 전달 여부 선택
  2. appPath : subworkflow 목록창에서 대상 워크플로우 선택. 선택하면 workflowId, workflowName은 자동 설정됨
  3. workflowId : 워크플로우 ID
  4. workflowName : 워크플로우 이름
  5. configuration : key-value로 설정
  6. forceOK : 데이터 처리가 실패해도 정상으로 표시하고 종료

Example#

executeHive(create table as select 쿼리 실행)노드가 실행되는 워크플로우(G1378)를 subWorkflow로 설정 후 실행 및 결과까지 확인한다.

  1. executeHive workflow 생성
create table air_pollution_subworkflow_test              as select * from air_pollution limit 10

  1. [subWorkflow]노드를 drag & drop 한 후 Property 항목을 입력한다. 1번에서 작성한 워크플로우를 찾아 등록한다.

  1. subworkflow 실행결과 executeHive노드에 설정한 쿼리가 정상 실행되었다.
