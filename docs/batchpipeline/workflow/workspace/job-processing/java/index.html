java

#Definition

HDFS상에 미리 올려둔 Java Class의 이름을 main Class에 작성해야 실행이 가능하다. [Flow구성]노드 중 [java]노드를 drag & drop 한 후 Property 항목을 입력한다. Property 패널의 [더보기+] 버튼을 누르면 입력가능한 전체 Property 항목을 볼 수 있다.

#Set

[setting], [scheduler], [parameter] 설정은 [워크플로우 생성] > [설정]을 참고한다.

#property

[Node Description] 작성 중인 노드명 입력

  1. prepare : 노드 실행 결과를 HDFS에 저장하기 위해 파일삭제/폴더생성 작업 선 수행(워크플로우 반복실행 시 활용가능)
    • delete : 노드 실행 전 삭제할 폴더/파일 경로
    • mkdir : 노드 실행 전 생성할 폴더 경로
  2. file : 노드에서 사용할 라이브러리 경로를 설정
  3. archive : archive경로를 설정
  4. retry
    • max : 실행 실패 시 재시도 횟수
    • period : 재시도 주기(분 단위)
  5. property : 실행에 사용할 property (key, value) 입력
  6. javaOpts : JAVA 실행시 적용할 옵션 (ex : -Xms256m)
  7. captureOutput : JAVA 실행결과를 다음 노드로 전달 여부
  8. mainClass : 실행할 JAVA 클래스 입력
  9. params : main 함수에 전달할 parameter 입력
  10. jobXml : jobXml 경로 입력. 잡에 전달할 프로퍼티를 별도 xml 로 작성하여 전달가능
  11. forceOK : 데이터 처리가 실패해도 정상으로 표시하고 종료

#Example

AddTwoNumbers.class 파일을 java 노드에서 실행하는 예제이다.

  1. HDFS브라우저에서 AddTwoNumbers.class 파일을 먼저 등록한다.
  2. [Flow구성] > [java] 노드를 drag & drop 한 후 setting 패널에서 실행클러스터를 선택한다.
  3. property 패널에서 아래와 같이 입력한다.
    • 2.file : AddTwoNumbers.class 파일 저장경로 지정
    • 8.mainClass : mainClass명 입력 "AddTwoNumbers"
