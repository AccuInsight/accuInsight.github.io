Tutorial

개요#

Batch Pipeline으로 주어진 데이터를 처리하는 워크플로우를 작성한다.
VOD사용이력, 사용자 정보, product 정보 파일로 ETL작업을 진행하고, 최종 dataset은 HDFS에 저장한다. HDFS에 저장한 결과파일은 ML Modeling에 사용한다.

튜토리얼 수행을 위한 사전 준비사항은 다음과 같다.

사전 준비사항#

클러스터#

DHP에서 튜토리얼 수행용 클러스터를 만든다.

  1. 데이터노드 개수 최소 2개 이상
  2. Yarn Container's Mem Size 최소 9G 이상

데이터 준비#

ICOS에 업로드된 데이터를 사용한다(아래 파일 3가지를 직접 ICOS에 업로드하여 사용 가능).

  • products_demo.csv
  • users_demo.csv
  • vod_demo.csv

ICOS 제품 구매처 링크는 아래와 같다.

https://console.cloudz.co.kr/manage/SL/5

워크플로우 생성#

데이터파일 3개를 조인하여 Batch Pipeline에서 워크플로우를 생성한다. 상세 작업절차는 다음과 같다.

ICOS에서 파일 불러오기 → 3개의 파일 join → 사용할 column만 선택 → null값 제거→ column명 rename → HDFS에 저장

생성 메뉴로 이동#

Batch Pipeline에서 [워크플로우] > [생성]메뉴로 이동한다.

데이터 로딩#

사전 준비한 데이터를 클러스터에 로딩한다.

  • 좌측의 '데이터불러오기' 클릭
  • 데이터 불러오기 그룹이 열리면 ICOS 불러오기 노드를 캔버스 영역에 drag & drop하여 node 생성
  • property 패널 입력
  • Node name을 product정보 불러오기로 변경
  • file에 있는 브라우저 아이콘 클릭

  • ICOS File browser popup에서 데이터가 존재하는 스토리지와 버킷을 각각 선택
  • 디렉토리에서 업로드 할 'product_demo.csv' 파일을 선택한 뒤 확인 버튼 클릭

  • 우측 하단의 더보기 버튼을 클릭

  • Property 패널의 6. schema 우측의 '자동파싱 열기' 아이콘 클릭

  • 자동파싱 팝업에서 파싱된 샘플 데이터를 확인 후 '확인' 버튼 클릭
  • sample data / column info / formatted data 탭을 각각 클릭하여 데이터 확인 , 컬럼 데이터타입 및 컬럼명 변경 등 가능
  • 위와 같은 방법으로 user정보 불러오기와 vod불러오기 노드생성을 추가로 진행

vod불러오기 노드의 경우 아래를 참고하여 컬럼명을 변경한다.

  • 자동파싱 팝업에서 column info 탭 클릭
  • 자동파싱된 컬럼명을 변경하여 아래와 같은 순서로 입력
    • vod_ts
    • vod_ip
    • vod_url
    • vod_swid
    • vod_city
    • vod_country
    • vod_id

datajoin 노드를 활용하여 3개 파일 조인#

[데이터 처리하기]의 [datajoin]노드를 사용하여 product정보와 vod정보를 조인한다.

  • 좌측의 데이터 처리하기 클릭
  • 열린 패널에서 dataJoin 노드 우측 캔버스에 drag & drop하여 생성
  • product정보 노드에서 dataJoin 노드 방향으로 연결
  • vod정보 노드에서 dataJoin 노드 방향으로 연결
  • 우측 property패널에서 아래와 같이 입력
    • col1: products_demo_url 선택
    • col2: vod_url 선택
    • how : inner선택

[datajoin]노드를 추가하여 앞서 생성한 join노드와 user정보를 조인한다.

  • dataJoin 노드 drag & drop하여 1개 더 생성
  • 이전에 생성한 dataJoin 노드에서 신규로 생성한 dataJoin노드로 연결
  • user정보 노드에서 신규로 생성한 dataJoin 노드로 연결
  • 우측 property 패널에서 아래와 같이 입력
    • col1 : vodswid_df로 시작하는 컬럼 선택
    • col2 : swid 선택
    • how : outer 선택

필요 컬럼 선택#

[데이터 처리하기]의 [select]노드를 사용하여 분석에 필요한 컬럼만 선택한다.

  • select 노드 drag & drop하여 생성
  • 2번째로 생성한 dataJoin노드에서 select노드로 연결
  • 우측 property 패널 col checkbox에서 아래 5가지 column 체크
    • products_demo_title_df…
    • vod_ts_df…
    • vod_url_df…
    • gender_cd_df…
    • age_df…

column명 뒤에 붙는 데이터프레임명은 노드생성시 자동으로 생성되어 예시와 다를 수 있으므로 '…'으로 표기함

데이터 정제#

[데이터 처리하기]의 [dropna]노드를 사용하여 null값이 포함된 행을 제거한다.

  • dropna 노드 drag & drop하여 생성
  • select노드에서 dropna노드로 연결
  • 우측 property 패널
      1. how : any 선택

컬럼명 변경#

[데이터 처리하기]의 [withColumnRenamed]노드를 사용하여 컬럼명을 변경한다.

  • withColumnRenamed노드 drag&drop하여 생성
  • dropNa 노드에서 withColumnRenamed노드 방향으로 연결
  • 우측 property 패널에서 플러스 아이콘을 클릭하여 총 5개의 컬럼입력셋을 생성
  • column1개를 선택한 뒤 새로운 컬럼명을 입력
    • vodts_df_ckQ3_df… → ts
    • productsdemo_title_df…→ title
    • vod_url_df… → url
    • gender_cd_df… → gender
    • age_df … → age

HDFS에 저장#

워크플로우 실행결과를 HDFS에 저장한다. [데이터 내보내기]의 [HDFS내보내기] 노드를 사용한다.

  • 좌측 데이터 내보내기 클릭
  • HDFS내보내기 노드 drag & drop 하여 생성
  • withColumnRenamed 노드에서 HDFS 내보내기노드로 연결
  • 우측 property 패널
    • path의 browse 아이콘을 클릭하여 열리는 팝업에서 저장할 디렉토리 위치 선택 후 확인 클릭(ex. /tmp)
    • file에 생성할 디렉토리명 입력 (ex.modeling_data)

파이선코드 생성#

Batch Pipeline에서 작성한 워크플로우는 파이선코드로 생성할 수 있다.

  • ETL 영역안에 빈 영역 아무곳이나 클릭
  • 우측의 property 패널
    • appName입력 : test
    • source의 오른쪽 아이콘 클릭하여 파이선코드 자동생성
    • source의 왼쪽 아이콘 클릭

저장 및 실행#

Batch Pipeline에서 작성한 워크플로우를 저장 및 실행한다.

  • 우측 상단 [저장]버튼을 클릭한다. 워크플로우를 저장할 수 있다.
  • 우측 상단 [실행]버튼을 클릭한다. Batch Pipeline에서 작성한 워크플로우를 실행할 수 있다.

워크플로우 상태 확인#

워크플로우 실행 상태를 확인한다.

  • workflow 관리 화면으로 이동하여 상태 확인
  • 이미 성공으로 종료되었거나 오류가 발생하여 종료된 경우 INACTIVE 상태
  • 상세정보 컬럼의 Instance목록 아이콘 클릭

실행결과 확인#

워크플로우 실행 결과를 확인한다. 정상적으로 실행되었다면, 아래 경로에 결과파일이 등록되었음을 확인할 수 있다.

  • CONFIGURATION > HDFS BROWSER 메뉴로 이동
  • 지정했던 HDFS directory위치로 이동하여 내보내기로 생성한 디렉토리 하단의 part-00000파일 클릭
  • 우측 상단의 다운로드 버튼 클릭하여 파일내용 확인
