#생각하면서 일 하기

서비스 페이지를 만들기 위해서

  1. 다양한 사람들과 얘기하면서 의견을 주고받아라

    내가 진행 하고 있는 일에 대해서 말하고 다녀야한다, 꾸준히, 많은 사람들에게 피드백을 받고 아이디어를 Develop 할 수 있다.

  2. 설계부터 잘하고 들어가야함 (S/W Engineer의 역할)

    프로젝트 구조 directory 구조에 대해서 잘 생각해보고...

  3. 무엇을 할 수 있는지 무엇을 할 수 없는지에 대해 사전 Research를 하면서 파악을 해놔야 회의를 진행 할 때, 다른 팀원들에게 확답을 줄 수 있음. 따라서, 어떤 Framwork를 쓸지 각각의 장단점이 무엇인지도 조사해보자. 최종적으로 어떤 툴을 쓸지도 고려하는 것 까지 가야할 듯 / Mix해서 Develop 할 수도

    ex) Angular / Node / React / jQuery / Vue ...

    #Angular

    웹 Application Framework

    개발/테스트 환경 단순화 JQuery나, Js UI Component를 쉽게 재사용가능
    양방향 데이터 바인딩 -> 불필요 코드제거 (유지보수 쉬움)

    #Node.js

    이벤트 처리 I/O FrameWork

    비동기 프로그래밍(이벤트 요청 후 결과값을 기다리지 않아도 됨 -> 다양한 요청 처리 가능)
    백엔드 개발

    #React.js

    UI를 만들기 위한 Library -- 현재 쓰고있는 Docusaurus가 React기반입니다.

    Custom tag, 가상 Dom, 단방향 데이터 바인딩
    단일 페이지 응용프로그램, 모바일 응용프로그램등에 쓰임
    속도, 단순성, 확장성 제공목표

    #JQuery

    HTML의 클라이언트 사이트 조작을 단순화한 크로스 플랫폼 Library

    코드보기, 문서객체모델찾기, 애니메이션만들기, 이벤트제어, Ajax개발
    쉽게 디자인 가능
    개발자가 Plugin을 개발해야함

    #Vue

    UI빌드를 위한 오픈소스 프로그레시브 Framework, Angular + React의 장점을 합침

    다른 Js라이브러리 프로젝트에 통합하기 쉬움
    웹개발 단순화/정리를 위해 개발
    가장 대중적인 Front FrameWork
    익히기 쉬움
    Angular의 단순함 + React의 강력함 -> Vue
