subWorkflow

Definition#

다른 워크플로우를 하나의 Node로 실행 할 수 있습니다. 목록 열기를 누르면 목록이 팝업 되고 이중 하나를 선택하면 워크플로우 번호가 입력 됩니다. 좌측 [Flow구성]노드 중 [subWorkflow]노드를 drag & drop 한 후 Property 항목을 입력합니다. Property 패널의 [더보기+] 버튼을 누르면 입력가능한 전체 Property 항목을 볼 수 있습니다.

Set#

[setting], [scheduler], [parameter] 설정은 [워크플로우] > [생성] > [기본구성]을 참고합니다.

property#

[Node Description] 작성 중인 노드명 입력

  1. propagateConfiguration : subWorkflow 설정 전달 여부 선택
  2. appPath : "목록열기" 버튼 클릭하여 오픈한 Subworkflow 목록창에서 대상 워크플로우 선택 (선택한 워크플로우에 대해서 "subWorkflow 편집 바로가기" 버튼을 클릭하면 해당 워크플로우를 새창으로 오픈 가능)
  3. workflowId : 2.appPath 에서 선택한 워크플로우의 ID
  4. workflowName : 2.appPath 에서 선택한 워크플로우의 이름
  5. configuration : key-value로 설정
  6. forceOK : 실패 시 강제 OK 처리 여부

Example#

executeHive(create table as select 쿼리 실행)노드가 실행되는 워크플로우(G1378)를 subWorkflow로 설정 후 실행 및 결과까지 확인합니다.

  1. [executeHive] workflow 생성 
    ```create table air_pollution_subworkflow_test              as select * from air_pollution limit 10```
  2. 작성한 워크플로우 등록
  3. subworkflow 실행결과 executeHive노드에 설정한 쿼리 정상 실행 확인
