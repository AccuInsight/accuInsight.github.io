query

Definition#

RDBMS에 JDBC방식으로 접속하여 쿼리를 실행합니다.
좌측 [Flow구성]노드 중 [query]노드를 drag & drop 한 후 Property 항목을 입력합니다. Property 패널의 [더보기+] 버튼을 누르면 입력가능한 전체 Property 항목을 볼 수 있습니다.

Set#

[setting], [scheduler], [parameter] 설정은 [워크플로우] > [생성] > [기본구성]을 참고합니다.

property#

[Node Description] 작성 중인 노드명 입력

property 중 2~4 항목은 [연결관리]에 등록된 JDBC 정보 선택 시 자동으로 입력되며, 원하는 JDBC 목록이 없을 경우 [설정] > [JDBC 관리]에서 필요 JDBC를 먼저 등록합니다. [연결테스트] 아이콘을 선택하여 JDBC 연결설정 테스트를 할 수 있습니다.

  1. query : 실행할 Query 입력
    • [Query Editor]를 선택해서 팝업된 editor를 통해 코딩 가능
  2. username : [설정] > [JDBC 관리]에 저장된 JDBC 사용자ID
  3. password : [설정] > [JDBC 관리]에 저장된 JDBC 사용자 비밀번호
  4. jdbcUrl : [설정] > [JDBC 관리]에 저장된 JDBC URL정보
  5. variables
    • variable : 변수 명
    • value : 변수에 할당할 값
  6. forceOK : 실패 시 강제 OK 처리 여부

Example#

Maria DB에 접속 후 user 정보를 확인하는 쿼리를 수행하는 예제입니다. [설정] > [JDBC 관리]에 등록된 접속정보를 [연결관리] > [목록열기]를 통해 선택합니다. JDBC 관리에 등록한 정보가 없을 경우 [목록열기] 내 [신규생성]버튼을 통해 jdbc 정보를 새롭게 입력/저장할 수 있습니다.
