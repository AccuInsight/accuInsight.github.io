#Workflow 생성

Pipeline에서 제공하는 다양한 노드를 활용하여 Workflow를 구성한다.

분석대상 원천데이터를 [데이터불러오기], [데이터처리(기본, 고급)], [데이터내보내기]에서 제공하는 노드를 선형으로 연결하여 처리하는 것이 일반적인 패턴이나 [flow 구성]노드를 활용하여 아래와 같은 워크플로우도 구성 가능하다.

  • 데이터를 병렬로 처리하거나 선행작업의 결과에 따라 후행작업을 결정해야 하는 패턴
  • subworkflow 구성
  • 사용자가 직접 작성한 code를 실행하거나 머신러닝 학습을 위한 노드 제공

#생성화면 구성

Pipeline에서 [워크플로우] > [생성]화면으로 이동한다. 기본 구성은 다음과 같다.

① [그룹] 워크플로우가 포함된 그룹. 등록된 그룹명을 선택할 수 있고, 입력란에 직접 새로운 그룹명을 입력할 수 있다.

② [이름] 워크플로우 이름

③ [설명] 워크플로우에 대한 설명

④ [작업버튼] 워크플로우 생성/편집 작업에 필요한 버튼 모음

버튼명설명
목록에서가져오기저장된 워크플로우 목록을 팝업으로 띄우고, 선택한 워크플로우를 load 한다.
내보내기작성중인 워크플로우를 파일로 다운로드한다(파일명 : export_workflow_YYYY.MM.DD.json).
불러오기작성중인 워크플로우 저장여부 선택 후 json 파일로 저장한 워크플로우를 불러온다.
신규생성작성중인 워크플로우 저장여부 선택 후 Designer를 초기화한다.
저장작성중인 워크플로우를 저장한다.필수 입력항목이 비어있는 Component / Node는 붉게 처리되며, 입력 전까지 저장되지 않는다.
다른 이름으로 저장작성중인 워크플로우를 다른 이름으로 저장한다. 다른 이름으로 저장한 Component / Node의 UUID는 새로 발급되어 기존의 것과 중복되지 않는다.
삭제워크플로우를 삭제한다.
JSON 뷰어작성한 워크플로우의 JSON 형식을 확인한다.
세션 조회해당 클러스터의 세션 수행 이력을 조회한다(스냅샷/스키마 조회).
실행작성한 워크플로우를 실행한다.
정지실행한 워크플로우를 정지한다.

⑤ [Node] Pipeline에서 제공하는 데이터 탐색/전처리 작업을 위한 노드 모음으로, 필요한 노드를 선택하여 Designer에 Drog & Drop한다.

⑥ [Designer] 워크플로우를 생성할 수 있는 기본 화면. Pan 이동 및 마우스 휠을 통해 상하 이동이 가능하다.

⑦ [Component] 워크플로우 또는 특정 노드를 실행하기 위해 필요한 속성정보 입력도구. Setting, Scheduler, Parameter, Property 등 4개의 탭으로 구성되어 있고, 필수 입력항목(*)을 표기했다. [더보기+]버튼을 클릭하면 추가 입력할 수 있는 항목이 display된다.

⑧ [스케일] Designer에서 작성중인 워크플로우를 확대 또는 축소할 수 있는 도구. Ctrl + 마우스 휠 조작을 통해서도 화면 확대/축소가 가능하다.

⑨ [Select Project]

#데이터 스냅샷

데이터 불러오기, 데이터 샘플링, 데이터 처리하기(기본, 고급), 피처엔지니어링에 포함된 노드를 대상으로 데이터프리뷰 기능을 제공한다. 워크플로우 생성/편집 화면에서 노드 우측 상단 카메라 모양 버튼을 클릭한다.

속성정보를 정확히 입력했을 경우 선택한 노드 기준으로 [Data], [Overview], [Correlation] 탭을 제공한다.

#Data 탭

① 스냅샷버튼 선택시 [Data]탭을 기본 화면으로 불러온다.

② 컬럼명, 그래프, 데이터를 보여준다.

  • 컬럼명 : 컬럼명, 데이터타입, Data Range 및 group by 결과를 그래프로 보여준다. 컬럼순서대로 10개씩 보여주며, [>] 버튼으로 눌러 다음 순서의 10개 컬럼을 조회할 수 있다. 데이터는 오름차순정렬이 기본이며, 컬럼 우측 [▼] 버튼 선택시 내림차순으로 변경된다.
  • 그래프 : 컬럼별 group by 결과를 그래프로 보여준다.
  • 데이터 : 컬럼에 포함된 데이터를 그리드 형식으로 보여준다.

③ 기능별 상세내역은 다음과 같다.

  • Column Range : 데이터프리뷰에서 조회되는 컬럼정보를 선택한다.
  • 노드에 있는 스키마의 컬럼/데이터 개수를 보여준다.
  • 노드에 있는 데이터를 로컬로 다운로드 한다.
  • 데이터프리뷰 화면을 최소화한다.
  • 데이터프리뷰 화면을 종료한다.

#Overview 탭

노드에 포함된 데이터 개요, 컬럼별 요약정보 및 시각화 차트를 불러온다.

#Dataset Info

데이터셋의 중복값, 결측값과 그 비율을 보여준다.

#Columns

컬럼별 요약정보와 [Statistics], [Histogram], [Value] 정보를 제공한다.

  1. 컬럼별 요약정보
    • Distinct count, Missing, Unique, Outliers, Maximun, Mean, Minimun, Zeros 값과 전체 데이터에 대한 비율정보를 제공한다.

  1. Statistics
    • 분위수(quantile) 및 기술통계(descriptive statistics) 정보를 보여준다.

  1. Histogram
    • 선택된 컬럼에 대한 히스토그램을 보여준다.

  1. Value
    • 선택된 컬럼에 대한 Value정보를 보여준다.

#Correlation 탭

노드에 포함된 데이터의 상관분석 결과를 보여준다. 상관계수는 Pearson(기본값), Spearman에서 선택할 수 있으며, 전체 컬럼 중 최대 20개까지만 변수로 선택할 수 있다. Columns 드롭다운 버튼에서 선택컬럼을 변경할 수 있으며, [Draw] 버튼 클릭시 상관분석 결과가 refresh된다.

#스키마

데이터 불러오기, 데이터 처리하기에 포함된 노드에 한해 워크플로우를 디자인 하면서 스키마를 조회 할 수 있다. 워크플로우 생성/편집 화면에서 노드 우측 상단의 테이블 모양의 버튼을 클릭한다.

선택한 노드의 프로퍼티가 정확하게 입력되어 있으면 선택한 노드 시점의 스키마를 확인 할 수 있다.
