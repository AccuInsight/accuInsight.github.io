#프로젝트 관리

프로젝트는 논리 그룹으로 구성된 워크스페이스, 저장소, 배포 리소스를 그룹화하여 관리합니다.

  • 이름 : 프로젝트를 대표하는 제목
  • 설명 : 프로젝트에 대한 설명
  • 상태 : 현재 프로젝트의 상태 표기
  • 워크스페이스 : 생성된 워크스페이스 수 (목록 바로가기)
  • Auto ML : 수행된 Auto ML 수 (목록 바로 가기)
  • 소스저장소 : 저장소에 등록된 소스그룹 수 (목록 바로가기)
  • 배포저장소 : 저장소에 등록된 배포모델 수 (목록 바로가기)
  • 배포 : 배포 모델을 RESTful service로 배포한 항목 수 (목록 바로가기)
  • Owner : 프로젝트 생성 계정
  • Members : 프로젝트를 공유하는 계정 정보
  • 생성일 : 생성일, 시 표기
  • Action : "수정 / 삭제"

1) 프로젝트 생성

  • 우측 상단의 ‘프로젝트 생성’ 버튼을 눌러 프로젝트 생성
  • 프로젝트 이름 중복 생성 불가
  • 프로젝트 내 워크스페이스, 배포 컨테이너들의 최대 리소스(CPU, GPU, Memory) 지정
  • 프로젝트 공유 사용자 추가  : 우측의 전체 사용자 명단에서 공유하려는 계정을 선택하여 좌측의 프로젝트 사용자 명단으로 이동

2) 프로젝트 수정

  • 프로젝트 설명과 공유 사용자 정보만 변경 가능
  • Resource Quota의 경우 이전에 설정된 값보다 작은 값으로 변경 불가
