클러스터

클러스터 생성#

‘클러스터’ 메뉴를 클릭 후, 왼쪽 상단 ‘클러스터 생성하기’ 버튼 클릭 합니다.

기본 설정#

기본 정보, 생성방법, 소프트웨어 구성를 입력합니다.

  1. 하둡 서비스 명
    • 이름 : 생성할 클러스터의 이름으로 영문으로만 입력 가능
    • 설명 : 생성할 클러스에 대한 설명
  2. 생성방법 선택
    • 빠른시작 : 리소스가 최소 사양으로 자동 설정
    • 상세설정 : 각 eco 노드의 리소스를 세부적으로 설정
  3. 소프트웨어 구성
    • Chart : DHP에서 지원/제공하는 애플리케이션 번들로 선택 시 포함된 Hadoop eco 및 버전 확인 가능
    • Hadoop Eco : 클러스터 생성 시 포함할 Hadoop Eco 선택
    • Jupyter Notebook Password : Jupyter Notebook 생성 시 사용할 password 입력

상세 설정#

생성방법 선택에서 "상세설정"으로 선택 시 하드웨어, Storage 구성을 추가 입력합니다.

  1. 하드웨어 구성
    • Name Node 설정
      • Name Node CPU : Name Node pod에 할당할 CPU core 수
      • Name Node Memory : Name Node pod에 할당할 Memory(G)
    • Data Node 설정
      • Data Node Pod Count : Data Node의 pod 개수
      • Data Node CPU : Data Node pod에 할당할 CPU core 수
      • Data Node Memory : Data Node pod에 할당할 Memory(G)
    • Resource Manager 설정
      • Resource Manager CPU : Resource Manager pod에 할당할 CPU core 수
      • Resource Manager Memory : Resource Manager pod에 할당할 Memory(G)
    • Node Manager 설정
      • Node Manager Pod Count : Node Manager pod 개수
      • Node Manager CPU : Node Manager pod에 할당할 CPU core 수
      • Node Manager Memory : Node Manager pod에 할당할 Memory(G)
  2. Storage 구성
    • HDFS설정
      • Block Replication Factor : Block 복제 개수를 1~3 사이로 설정 (기본값 : 1)
    • Local Persistence Volume
      • Local Storage Enable : Local Storage 가능여부를 선택

클러스터 조회#

  1. 클러스터 상세 조회
    • 클러스트 명 클릭
    • 상세한 설명은 클러스터 상세 조회 참조
  2. 클러스터 권한 설정
    • 권한을 설정할 클러스터명 우측의 권한 버튼 클릭
    • 권한 버튼은 해당 클러스터 관리자 권한이 있는 사용자 또는 해당 클러스터에 기록 권한이 있는 경우 활성화
    • 상세한 설명은 클러스터 권한 설정 참조
  3. 클러스터 삭제
    • 클러스트명 우측의 삭제 버튼 클릭
    • 권한 버튼은 해당 클러스터 관리자 권한이 있는 사용자 또는 해당 클러스터에 기록 권한이 있는 경우 활성화
  4. 노드 수정
    • 변경할 노드의 노드 수정 버튼 클릭
    • 상세한 설명은 클러스터 노드 수정 참조

클러스터 권한 설정#

  1. 현재 선택한 클러스터에 대한 권한을 가지고 있는 사용자
  2. 현재 로그인한 사용자와 동일 User Group인 사용자
  3. 선택한 사용자에 대해서 권한을 설정
    • 조회 : DHP 포함 타 서비스 이용 시 해당 클러스터 조회 가능
    • 실행 : 해당 클러스터에 타 서비스 작업을 수행 가능
    • 기록 : 해당 클러스터의 수정/삭제 및 타 사용자에 권한 부여 가능
  4. 변경된 권한 정보를 저장

클러스터 노드 수정#

노드 변경 창에서 변경 개수를 입력 후 변경 버튼을 클릭하면, 해당 노드의 개수가 입력한 변경 개수로 증가/감소 하게 됩니다.

클러스터 상세 조회#

OVERVIEW#

클러스터 요약 정보로 클러스터 설명, 생성 정보, 하드웨어 구성를 확인 할 수 있습니다.

SERVICE STATUS#

Hadoop Eco 서비스 상태를 확인 할 수 있습니다.

APPL HISTORY#

클러스터에서 수행된 어플리케이션의 이력을 확인 할 수 있으며, 하단 어플리케이션 ID 클릭 시 수행 이력을 상세하게 확인 할 수 있습니다.

MONITORING#

클러스터에 CPU와 메모리 사용률, HDFS Disk, Namenode Heap Memory를 모니터링 할 수 있습니다.
