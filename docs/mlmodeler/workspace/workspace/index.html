Workspace

Workspace는 머신러닝 분석에 필요한 알고리즘과 데이터셋을 선택하여 UI기반의 파이프라인을 구성하고 학습까지 수행할 수 있는 작업공간이다. 앞서 설명한 바와 같이 SparkML에서 제공하는 머신러닝 알고리즘을 선택할 수 있다.

  • Create New Pipeline(신규) 버튼 클릭 시 새로운 Pipeline이 생성되고 탭이 추가된다.
  • Load Pipeline(불러오기) 버튼 클릭 시, 저장된 Pipeline리스트를 가져올 수 있는 Load Pipeline창이 열린다.
  • 파이프라인 안에 Select Dataset폴더 버튼 클릭 시, 저장된 Dataset리스트를 가져올 수 있는 Load Dataset 창이 열린다.
  • Tab옆 x 버튼 클릭 시 탭이 닫힌다.

Pipeline (파이프라인) 설정#

학습시킬 Pipeline을 설정하는 방법은 새로운 Pipeline을 생성하거나 저장된 Pipeline을 가져오는 2가지 방법이 있다.

Pipeline 생성#

새로운 Pipeline을 생성한다.

  • New(신규) 버튼을 클릭한다.
  • 탭이 추가되며 새로운 Pipeline을 생성할 수 있다.

Dataset(데이터셋) 불러오기#

학습시킬 Dataset을 불러온다.

  • Workspace pipeline의 첫Item은 Dataset불러오는 용도이다. 폴더 버튼을 클릭하면 Load dataset창이 나타난다.
  • Dataset Name을 입력 후 Search버튼 클릭 시 해당 키워드를 포함한 Dataset이 검색된다.
  • 선택할 Dataset의 Apply아이콘 클릭 시 Dataset이 Load된다.

Pipeline구성 및 아이템 속성 설정#

학습할 알고리즘을 선택하고, parameter(파라미터)을 설정한다.

  • ‘+’버튼 클릭 시 Item(알고리즘)이 추가된다.
  • ‘X’버튼 클릭 시 Item이 삭제된다.(최하단 Item부터 순차적으로 삭제 가능)
  • Algorithm 클릭 시 우측에 해당 Algorithm의 파라미터 수정 화면으로 변경된다.
  • Algorithm 설명을 참고해서 Algorithm에서 사용되는 파라미터를 적절히 조정한다.
  • property항목 중 Automodel (지원되는 알고리즘에 한함)을 true로 변경하면 선택한 알고리즘 내에서 최적의 파라미터를 찾아준다.
  • 파라미터 항목 중 validationSetPath입력 시 splitWeight param이 무시되고, 전체 data가 학습으로 쓰이며, path의 데이터로 모델을 평가한다(단, data 스키마는 입력 dataset과 동일해야 함).

Pipeline저장#

앞서 작업한 Pipeline을 저장한다.

  • Pipeline작성 후 우측 상단Pipeline(파이프라인) 저장 버튼을 클릭한다.
  • Save Pipeline팝업에서 저장할 Pipeline이름과 설명 작성 후 Save버튼을 클릭하면 작성한 Pipeline이 저장된다.
  • 이미 저장이 되어있는 Pipeline인 경우에는 이름, 설명을 수정(Save버튼)하거나 새로운 Pipeline으로 등록(Save As)한다.

Pipeline불러오기#

저장된 Pipeline을 가져온다.

  • 좌측 상단 Load Pipeline(불러오기) 버튼을 클릭한다.
  • Pipeline Name에 검색어를 입력 후 검색 버튼을 클릭하거나 날짜를 선택하면 해당 Pipeline이 조회된다.
  • 팝업 창에서 Pipeline리스트 중 하나를 선택하고 Apply아이콘을 클릭한다.
  • 선택한 Pipeline이 Workspace에 나타난다.

학습 실행#

위에서 설정한 Pipeline을 실행한다.

  • Pipeline구성 후 우상단 Run(실행) 버튼을 클릭한다.

  • Cluster 항목 옆의 Browse 버튼을 통해 학습을 실행할 클러스터를 선택한다.

  • 학습된 모델을 저장하려면 Model 항목 옆의 Save 체크박스를 선택하고 모델명, 설명, 모델 저장 경로를 선택한 입력한다.

  • 학습 시 사용된 validation Set에 대한 예측 결과(validation)를 저장하려면 Dataset 항목 옆의 Save 체크박스를 선택하고 데이터셋명, 설명, 저장 경로, 저장할 컬럼목록을 입력한다. (컬럼의 순서는 사용자의 선택 순서를 따른다. split 된 데이터의 경우 test set만 저장 된다.[그 외의 알고리즘은 모든 row 저장])
  • Run을 눌러 학습을 시작하면 좌상단 Pipeline status 버튼이 Idle(대기)에서 Now Running으로 변경된다.

학습상태 조회#

학습 상태를 조회한다.

  • 좌상단 Run Status (대기, idle, Now Running 등) 버튼을 클릭하면 실행 중인 학습들의 상태가 나타난다.

학습 실행 취소 및 결과조회#

  • [취소] Pipeline Run후 학습이 Running상태일때 우상단 Cancel(취소) 버튼을 클릭하면 수행 중인 작업이 취소된다.
  • [결과조회] 학습이 완료되면 우측 상단의 Show results(결과조회) 버튼이 활성화 되며, 이를 클릭하여 학습 결과를 조회한다. 또는 좌측 상단 Run Status (대기, idle 등) 버튼을 클릭하면 나오는 Running Status창에서 “Results”로도 조회할 수 있다.

Training Result조회#

Training Result탭을 클릭하여 학습 결과(평가)를 조회한다.

  • 마지막 Item이 머신러닝 알고리즘일 경우, 테스트 셋에 대한 Evaluation 결과가 보여진다.
  • 알고리즘에 따라 보여지는 그래프가 다르다.

Model Description조회#

Model Description탭을 클릭하여 학습 결과(모델)을 조회한다.

  • 마지막 Item이 머신러닝 알고리즘일 경우, Model description클릭 시 학습된 모델 결과가 보여진다.

Generated Code 조회#

Generated Code탭을 클릭하여 Spark에서 실행한 Code를 조회한다.

  • 상단의 생성된 코드 보기 옆의 복사 아이콘을 클릭하여 Code를 복사하여 직접 Spark에서 실행해 볼 수 있다.
